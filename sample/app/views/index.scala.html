Play-Bootstrap3

Play Framework library for Bootstrap 3

This is a collection of input helpers and field constructors for Play Framework 2.3 to render Twitter Bootstrap 3 HTML code.

New 0.3-SNAPSHOT

Changes in helper's names: now in 0.3, the names and the structure of some helpers have been simplified. You can see the full changelog here but if you want to get this working with the new version you need to do the following:
  • For every static, submit, reset and actions helper within a formWithFC now need to include explicitly the corresponding field constructor For example:
    @@b3.submit('class -> "btn btn-default"){ Send } → @@b3.submit('class -> "btn btn-default"){ Send }(fc)
And then, replace the following (the first 2 ones in first place):
  1. b3.horizontal.form( → b3.form( (its important to add the ( to avoid replace formWithFC as well)
  2. b3.inline.form( → b3.form( (its important to add the ( to avoid replace formWithFC as well)
  3. b3.vertical.formWithFC → b3.vertical.form
  4. b3.horizontal.formWithFC → b3.horizontal.form
  5. b3.inline.formWithFC → b3.inline.form
  6. b3.clear.formWithFC → b3.clear.form
  7. b3.vertical.static → b3.static
  8. b3.horizontal.static → b3.static
  9. b3.horizontal.submit → b3.submit
  10. b3.horizontal.reset → b3.reset
  11. b3.actions → b3.free()
  12. b3.horizontal.actions → b3.free()

Features

Simple to use

It tries to follow the same rules as the helper library from Play Framework.

@code { @@import b3.vertical.fieldConstructor // Declares a vertical field constructor as default @@b3.form(routes.Application.login) { @@b3.email( loginForm("email"), '_label -> "Email", 'placeholder -> "example@@mail.com" ) @@b3.password( loginForm("password"), '_label -> "Password", 'placeholder -> "Password" ) @@b3.checkbox( loginForm("remember"), '_label -> None, '_text -> "Remember me", 'value -> true ) @@b3.submit('class -> "btn btn-default"){ Sign in } } }

There is a different field constructor for each kind of form. Normally, you will declare one of them as default for each of your templates, but you can mix them if you want.

Multifield support

Sometimes you may need two or more fields within the same line in a horizontal or vertical form (for a set of checkboxes, a date range, ...). With the b3.multifield you will be able to create your own custom helpers in a very easy way.

@bsExample { @b3.horizontal.form(routes.Application.index, "col-md-2", "col-md-10") { case (hfc, lang) => @b3.multiCheckbox( (fooForm("foo"), Map('_text -> "Foo")), (fooForm("bar"), Map('_text -> "Bar")), (fooForm("beer"), Map('_text -> "Beer")) )('_label -> "Checkboxes", 'class -> "multi-checkbox-list inline", '_help -> "Mark what you want")(hfc, lang) @b3.datepicker( fooForm("dateStart"), 'value -> "15-11-2014" )( fooForm("dateEnd"), 'value -> "31-12-2014" )('_label -> "Date range", "data-date-start-date" -> "10-11-2014", '_help -> "Select a date range from 10-11-2014" )(hfc, lang) } }{ @@b3.multiCheckbox( (fooForm("foo"), Map('_text -> "Foo")), (fooForm("bar"), Map('_text -> "Bar")), (fooForm("beer"), Map('_text -> "Beer")) )('_label -> "Checkboxes", 'class -> "multi-checkbox-list inline", '_help -> "Mark what you want") @@b3.datepicker( fooForm("dateStart"), 'value -> "31-10-2014" )( fooForm("dateEnd"), 'value -> "31-12-2014" )('_label -> "Date range", "data-date-start-date" -> "01-01-2014", '_help -> "Select a date range from 10-11-2014" ) }

Note: these two helpers (b3.multiCheckbox and b3.datepicker) are not part of the library, but here it is explained how to easily implement them (or just copy them).

Coherence and consistency

Boolean attributes

Sometimes the HTML syntax could be confusing. For example, all of the following codes are equivalent:

@code { }

For that reason, b3 helpers will simply remove the attribute with false values and put "true" value for true ones. So you can do:

@code { @@b3.text( fooForm("foo"), '_label -> "A maybe disabled text", 'disabled -> maybeTrueFunction(...) ) }

Readonly attribute for checkboxes, radio buttons and selects

Another frustrating and typical situation occurs when you try to use the readonly attribute for checkboxes, radio buttons or selects. It isn't implemented, so you usually need to add an extra <input type="hidden"> if you need to send its value within your POST request. For that cases, b3 helpers for checkboxes, radio buttons and selects automatically add that input for you when the attribute readonly is present.

Additionally, in that cases, when you add the readonly attribute, b3 helpers will also add the disabled attribute. Then, Bootstrap 3 will avoid their interactability.

Installation

You only need to add it as a dependency in your build.sbt file. It is cross compiled against Scala 2.10.4 and 2.11.0.

@code { libraryDependencies ++= Seq( "com.adrianhurt" %% "play-bootstrap3" % "0.1.1" ) }

If you want to use the latest snapshot, add the following instead:

@code { resolvers += "Sonatype OSS Snapshots" at "https://oss.sonatype.org/content/repositories/snapshots/" libraryDependencies ++= Seq( "com.adrianhurt" %% "play-bootstrap3" % "0.3-SNAPSHOT" ) }

Note the play-bootstrap3 has Bootstrap 3 and jQuery as WebJar dependencies, so they are included automatically. Specifically, it adds the following dependencies for you:

@code { "org.webjars" % "jquery" % "2.1.1" "org.webjars" % "bootstrap" % "3.3.1" }
